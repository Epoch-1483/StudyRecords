# 线程池核心要点（编译自 Vault2）

> 本页由 LLM Wiki 专家在知识查询后归档，综合自多线程相关笔记，便于复习与关联。

## 为什么需要线程池
传统方式"用到线程时创建、用完即销毁"，频繁创建/销毁开销大（见 [[线程池]]）。线程池复用线程：任务提交时若池中没有空闲线程且无法新建，则进入**任务队列排队**。

## 线程池的组成：核心线程 vs 临时线程
- **核心线程**：常驻，池子保有的基本线程数。
- **临时线程**：当核心线程都在忙、且任务队列已排满时，才创建临时线程处理溢出任务；空闲超过存活时间后回收。
- 触发临时线程的时机：排队排满且仍有新任务进来（见 [[线程池]]）。

## ThreadPoolExecutor 七大参数
```java
ThreadPoolExecutor pool = new ThreadPoolExecutor(
        3,                                      // 1. corePoolSize 核心线程数量
        6,                                      // 2. maximumPoolSize 最大线程数(≥核心)
        60,                                     // 3. 空闲线程最大存活时间
        TimeUnit.SECONDS,                       // 4. 时间单位
        new ArrayBlockingQueue<>(3),            // 5. 阻塞任务队列
        Executors.defaultThreadFactory(),       // 6. 线程工厂
        new ThreadPoolExecutor.AbortPolicy()    // 7. 拒绝策略(直接抛异常)
);
```
（来源：[[线程池]]）

## 线程池大小怎么定
- 获取 CPU 核数（最大并行数）：`Runtime.getRuntime().availableProcessors()`。
- 经验：CPU 密集型 ≈ 核数；IO 密集型可明显更大（来源：[[线程池]]）。
