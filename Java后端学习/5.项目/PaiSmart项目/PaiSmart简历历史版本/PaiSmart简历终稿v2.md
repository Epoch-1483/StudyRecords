## PaiSmart 简历终稿（问题→方案→效果 格式）

**项目名称：PaiSmart — 企业级 RAG 智能知识库系统**
**GitHub：** https://github.com/Epoch-1483/PaiSmart
**时间：** 2025.04 — 2026.05
**技术栈：** Spring Boot 3.4、Elasticsearch 8、Kafka、Redis、MinIO、MySQL、WebSocket、Vue 3、TypeScript

**项目介绍：** 面向企业级文档管理与知识问答场景，搭建从文档上传、智能解析切块、向量化存储到混合检索与流式对话的全链路 RAG 系统，支持多租户数据隔离与 Token 计量计费。

**核心职责：**

1. **异步文档流水线**：上传与解析耦合导致接口阻塞，大文件中断后无法续传。基于 Kafka 构建"合并→解析→切块→向量化→入库"异步流水线，事务投递保证一致性，死信队列兜底失败消息；分片上传采用 SparkMD5 指纹去重 + Redis BitMap O(1) 状态判定，实现断点续传与秒传校验。

2. **多路检索召回**：单一向量检索漏召回关键词，单一关键词检索丢失语义，且多租户场景下需防止越权访问。设计 ES 混合检索索引（2048 维 dense_vector + IK 分词器），KNN 宽召回（k=topK×30）+ BM25 Rescore 精排，bool.filter 注入权限条件实现检索级租户隔离；解析层 LiteParse OCR + Apache Tika 多引擎适配，切块采用"段落→句子→HanLP 分词"三级粒度 + 语义 overlap 防止断裂。

3. **流式对话架构**：传统 LLM 单轮回复无法调用知识库，长响应 HTTP 请求易超时。构建 ReAct Agent 架构（最大 4 轮推理、8 次工具调用），LLM 自主决策调用 search_knowledge 等工具；WebSocket 推送 JSON 帧（start/chunk/tool_call/completion）实现打字机式流式对话，内置心跳保活、自动重连与断线快照恢复。

4. **多租户隔离**：多租户场景需数据严格隔离，且 Token 用量需精确计量防超发。基于 Spring Security + JWT + RBAC 构建三层数据隔离（私有/组织/公开），JWT 5 分钟窗口自动续签 + Redis 黑名单注销；Token 计量采用 Redis 原子计数预扣-结算模式，按实际消耗扣费。
