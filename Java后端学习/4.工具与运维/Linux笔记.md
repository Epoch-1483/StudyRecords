**为什么学习Java后端开发也要学习Linux呢？：**

1. **生产环境主流操作系统**：绝大多数 Java 后端服务部署在 Linux 服务器上（如 CentOS、Ubuntu），因为其稳定、高效、开源且资源占用低。
2. **部署与运维需求**：你需要通过 Linux 命令启动/停止服务、查看日志（如 `tail -f`）、监控进程（`ps`, `top`）、管理端口（`netstat`, `ss`）等。
3. **环境配置**：安装 JDK、Tomcat、MySQL、Redis、Nginx 等中间件通常在 Linux 上进行，需熟悉路径、权限、环境变量等操作。
4. **调试与排错**：线上问题排查（如内存溢出、CPU 飙高）依赖 Linux 工具（`jstack`, `jmap`, `grep`, `awk`, `vim` 等）。
5. **自动化与 DevOps**：CI/CD 脚本（Shell）、Docker、Kubernetes 等都基于 Linux 环境运行。

# 8. Linux 命令

## 8.1. Linux 命令基础

**什么是命令、命令行？**

学习Linux，本质上是学习在命令行下熟练使用Linux的各类命令。

- 命令行：即 Linux 终端，可以提供字符化的操作页面供命令执行。

- 命令：即 Linux 操作的指令，是相同内置的程序，可以以字符化的形式去使用。

**Linux 命令基础格式**

### 8.1.1. Linux 命令通用格式

无论是什么命令，用于什么用途，在 Linux 中，命令有其通用的格式：

```
command [-options] [parameter]
```

- **command**：命令本身
- **-options**：[可选，非必填] 命令的一些选项，可以通过选项控制命令的行为细节
- **parameter**：[可选，非必填] 命令的参数，多数用于命令的指向目标等

**说明**：语法中的 `[ ]` 表示可选的意思。

---

### 8.1.2. 示例

1. `ls -l /home/user`

- `ls` 是命令本身
- `-l` 是选项
- `/home/user`是参数
- **含义**：以列表的形式显示 `/home/user`目录内的内容

2. `cp -r test1 test2`

- `cp` 是命令本身
- `-r` 是选项
- `test1` 和 `test2` 是参数
- **含义**：复制文件夹 `test1` 成为 `test2`

---

## 8.2. Linux 命令入门

### 8.2.1. **ls 命令**

**作用**：  
`ls` 命令用于列出目录下的内容。

**语法格式**：

```
ls [-a -l -h] [Linux路径]
```

- `-a`、`-l`、`-h` 是可选的选项：

- `-a`：显示所有文件，包括隐藏文件（以 `.` 开头的文件）
- `-l`：以长列表形式显示文件详细信息（如权限、所有者、大小、修改时间等）
- `-h`：以人类可读的形式显示文件大小（如 KB、MB）

- `[Linux路径]` 是可选的参数，表示要查看的目录路径。若不指定，则默认为当前工作目录。

**说明**：  
当不使用任何选项和参数，直接执行 `ls` 命令时，表示：以平铺形式（简洁列表）列出当前工作目录下的内容。

![](../../图片/3.默认图片/1772794926235-7d6871af-22f4-4f9a-939f-deeb744ff5c9.png)

## 8.3. **HOME目录和工作目录**

**问题**：  
直接输入 `ls` 命令，表示列出当前工作目录下的内容。那么，当前工作目录是什么？

---

### 8.3.1. **Linux系统命令行终端的默认行为**

在启动时，Linux系统的命令行终端会默认加载以下设置：

- **当前登录用户的 HOME 目录** 被设为 **当前工作目录**。  
    因此，执行 `ls` 命令时，显示的是该用户的 HOME 目录内容。

---

### 8.3.2. **HOME目录说明**

- **定义**：每个 Linux 操作用户在系统中都有一个个人账户目录，称为 HOME 目录。
- **路径格式**：`/home/用户名`

- 示例：图中用户为 user，其 HOME 目录为：`/home/user`

- **备注**：Windows 和 Linux 系统都设有用户的 HOME 目录（如 Windows 中的 `C:\Users\用户名`）。

---

### 8.3.3. **示例输出**

```
[user@bogon ~]$ ls
Desktop  Documents  Downloads  Music  Pictures  Public  Templates  Videos
```

- 此输出是 `ls` 命令的结果，显示的是当前用户 `itheima` 的 HOME 目录内容。
- `~` 符号代表当前用户的 HOME 目录。

---

**总结**：

- 启动终端后，默认工作目录是当前用户的 HOME 目录。
- 执行 `ls` 不带参数时，实际查看的是 `/home/用户名` 下的内容。
- `~` 是 HOME 目录的快捷符号。

---

## 8.4. Linux 命令的参数和选项

### 8.4.1. **ls 命令的选项与参数说明**

刚刚展示了直接使用 `ls` 命令，未使用任何选项和参数。

**命令格式回顾**：

```
ls [-a -l -h] [Linux路径]
```

那么，`ls` 的选项和参数具体有什么作用呢？我们先来看**参数**部分。

---

#### 8.4.1.1. **参数的作用**

- **当 ls 不使用参数时**：  
    表示列出 **当前工作目录** 的内容，即用户的 **HOME 目录**。  
    （例如：用户为 user，则显示 `/home/user`下的内容）
- **当使用参数时**：  
    `ls` 命令的参数表示指定一个 Linux 路径，用于列出该路径下的内容。例如：

```
ls /
```

将列出 `/` 目录下的文件和子目录。

![](../../图片/3.默认图片/1772795672614-1d5ff98b-4216-4c7a-90a5-047f8faab1ff.png)

---

**总结**：

- 参数是可选的，用于指定要查看的目录路径。
- 若不指定参数，默认查看当前工作目录（通常是用户的 HOME 目录）。
- 使用参数可以灵活地查看系统中任意目录的内容。

### 8.4.2. **ls 命令的** `**-a**` **选项**

#### 8.4.2.1. **语法说明**

`ls` 命令支持使用选项，其通用语法为：

```
ls [-a -l -h] [Linux路径]
```

- `**-a**` **选项**：表示 "all"（全部），用于列出目录下的**所有文件和文件夹**，包括**隐藏的文件/文件夹**。

---

#### 8.4.2.2. **示例对比**

![](../../图片/3.默认图片/1772795837407-562feacc-1ebc-42db-908d-ac624005881d.png)

- 可以看到，`ls -a` 列出的内容比普通 `ls` 更多。
- 多出的以 `.` 开头的文件或目录是 Linux 系统中的**隐藏文件/隐藏文件夹**。

---

#### 8.4.2.3. **隐藏文件规则**

- 在 Linux 中，只要文件或文件夹名称**以** `**.**` **开头**，就会被系统自动隐藏。
- 常见隐藏文件示例：

- `.bash_history`：记录命令历史
- `.bash_profile`：用户登录时执行的配置文件
- `.config`、`.cache`：应用程序配置和缓存目录

---

#### 8.4.2.4. **关键点总结**

- 普通 `ls` 不显示隐藏文件。
- 只有通过 `-a` 选项，才能查看这些以 `.` 开头的隐藏文件/文件夹。
- `-a` 是查看完整目录结构的重要选项，尤其在调试或配置系统时非常有用。

---

### 8.4.3. **ls 命令的** `**-l**` **选项**

#### 8.4.3.1. **语法回顾**

```
ls [-a -l -h] [Linux路径]
```

- `**-l**` **选项**：表示以“列表”（竖向排列）的形式展示目录内容，并显示更多详细信息。

`**rm**` **命令中的通配符** `*****` **模糊匹配规则**：

- `*` 是 **通配符（wildcard）**，表示匹配**任意内容（含空字符串）**
- 常用于路径或文件名匹配，示例如下：

- `test*` → 匹配以 `test` 开头的文件/目录（如 `test.txt`, `test123`, `test`）
- `*test` → 匹配以 `test` 结尾的文件/目录（如 `a_test`, `mytest`, `test`）

- `*` 由 **Shell 解析**（非 `rm` 自身处理），在执行前已展开为实际路径列表；
- 若无匹配项，部分 Shell 会原样传递 `*test*` 给 `rm`，可能导致报错（如 `rm: cannot remove '*test*': No such file or directory`）；
- 使用时建议先用 `ls *pattern*` 预览匹配结果，避免误删。

### 8.10.3. find 命令 - 通配符

根据语法：`find 起始路径 -name "被查找文件名"`

被查找文件名，支持使用通配符 `*` 来做模糊查询。

- **符号** `*****`：表示通配符，即匹配任意内容（包含空）。

**示例说明：**

- `test*`：表示匹配任何以 **test** 开头的内容。
- `*test`：表示匹配任何以 **test** 结尾的内容。
- `*test*`：表示匹配任何包含 **test** 的内容。

基于通配符的含义，可以结合 `find` 命令做文件的模糊查询。

---

### 8.10.4. find 命令 - 按文件大小查找文件

**语法：**

```
find 起始路径 -size +|-n[kMG]
```

**参数说明：**

- `+`、`-`：分别表示“大于”和“小于”。
- `n`：表示具体的大小数值。
- `kMG`：表示大小单位，其中 `k`（小写字母）代表 KB，`M` 代表 MB，`G` 代表 GB。

**示例：**

- 查找小于 10KB 的文件：`find / -size -10k`
- 查找大于 100MB 的文件：`find / -size +100M`
- 查找大于 1GB 的文件：`find / -size +1G`

## 8.11. grep、wc 和管道符

### 8.11.1. grep 命令

可以通过 **grep** 命令，从文件中通过关键字**过滤文件行**。

**语法：**

```
grep [-n] 关键字 文件路径
```

**参数说明：**

- **选项** `**-n**`：可选，表示在结果中显示匹配的行的行号。
- **参数** `**关键字**`：必填，表示过滤的关键字。如果关键字带有空格或其它特殊符号，建议使用 `""`（双引号）将关键字包围起来。
- **参数** `**文件路径**`：必填，表示要过滤内容的文件路径，**可作为内容输入端口**。

---

### 8.11.2. wc 命令

可以通过 **wc** 命令统计文件的行数、单词数量等。

**语法：**

```
wc [-c -m -l -w] 文件路径
```

**选项说明：**

- **选项** `**-c**`：统计 bytes 数量（字节数）。
- **选项** `**-m**`：统计字符数量。
- **选项** `**-l**`：统计行数。
- **选项** `**-w**`：统计单词数量。

**参数说明：**

- **参数** `**文件路径**`：被统计的文件，**可作为内容输入端口**。

---

### 8.11.3. 管道符

学习了 `grep` 命令后，我们来学习一个新的特殊符号：**管道符（**`**|**`**）**。

**管道符的含义：**  
将管道符左边命令的结果，作为右边命令的输入。

**示例命令：**

```
cat itheima.txt | grep itheima
```

**执行过程解析：**

- `cat itheima.txt` 的输出结果（即文件内容）
- 作为右边 `grep` 命令的输入（被过滤的内容）

**命令输出：**

```
itheima is a brand of itcast.
```

## 8.12. echo、tail 和重定向符

### 8.12.1. echo 命令

可以使用 **echo** 命令在命令行内输出指定内容。

**语法：**

```
echo 输出的内容
```

**使用说明：**

- 无需选项，只有一个参数，表示要输出的内容。
- 如果内容复杂（例如包含空格或特殊符号），建议用 `""`（双引号）将其包围。

**演示：**

- **在终端上显示：Hello Linux**

```
[itheima@bogon ~]$ echo Hello Linux
Hello Linux
```

- **带有空格或** `**\**` **等特殊符号，建议使用双引号包围**

- **原因**：如果不包围，空格后的内容很容易被识别为第二个参数。尽管 `echo` 命令本身不受影响，但养成这个习惯对使用其他命令非常重要。

```
[itheima@bogon ~]$ echo "Hello Linux"
Hello Linux
```

---

### 8.12.2. 反引号 `

问题示例：`echo pwd`

```
[itheima@bogon ~]$ echo pwd
pwd
```

**问题分析：**  
本意是想输出当前的工作路径，但 `pwd` 被作为普通字符输出了，而不是执行 `pwd` 命令。

解决方案：使用**反引号（`）**

我们可以通过将命令用 **反引号**（通常也称之为 **飘号**）**`** 将其包围。

**作用：**  
被 **`** 包围的内容，会被作为命令执行，而非普通字符。

正确示例：

```
[itheima@bogon ~]$ echo `pwd`
/home/itheima
```

---

### 8.12.3. 重定向符

我们来学习两个特殊符号，**重定向符**：`>` 和 `>>`

- `**>**`：将左侧命令的结果，**覆盖**写入到符号右侧指定的文件中。
- `**>>**`：将左侧命令的结果，**追加**写入到符号右侧指定的文件中。

演示

- `**echo "Hello Linux" > itheima.txt**`

```
[itheima@bogon ~]$ echo "Hello Linux" > itheima.txt
[itheima@bogon ~]$ cat itheima.txt
Hello Linux
```

- `**echo "Hello itheima" > itheima.txt**`，再次执行，**覆盖新内容**。

```
[itheima@bogon ~]$ echo "Hello itheima" > itheima.txt
[itheima@bogon ~]$ cat itheima.txt
Hello itheima
```

- `**echo "Hello itcast" >> itheima.txt**`，再次执行，使用 `>>` **追加新内容**。

```
[itheima@bogon ~]$ echo "Hello itcast" >> itheima.txt
[itheima@bogon ~]$ cat itheima.txt
Hello itheima
Hello itcast
```

---

### 8.12.4. tail 命令

使用 `tail` 命令，可以查看文件尾部内容，跟踪文件的最新更改。

**语法：**

```
tail [-f -num] Linux路径
```

**参数与选项说明：**

- **参数** `**Linux路径**`：表示被跟踪的文件路径。
- **选项** `**-f**`：表示持续跟踪。
- **选项** `**-num**`：表示查看尾部多少行，不填默认 10 行。
